## Database Safety (CRITICAL)

Helper: `src/lib/db-guard.ts`

### Forbidden

- `supabase.insert(...)` — raw insert, not idempotent
- `safeInsert(...)` — unsafe in concurrent systems
- Generate ID via `Date.now()` — not unique

### Required

- Use `safeUpsert(table, payload, conflictKey)` for single-row writes
- Use atomic RPC (`untypedDb.rpc(...)`) for multi-table transactions
- All operations MUST be idempotent — running twice yields same result
- Check duplicate at application layer BEFORE hitting DB constraint

### Example

```tsx
import { safeUpsert } from '@/lib/db-guard';

// Single-row write
await safeUpsert({
  table: 'suppliers',
  data: { id: supplier.id, name: 'ABC', status: 'active' },
  conflictKey: 'id',
});

// Multi-table atomic operation — use RPC
await untypedDb.rpc('rpc_create_order', {
  p_header: headerData,
  p_items: itemsData,
});
```

## Supabase Client Usage

| Client      | When to use                                                |
| ----------- | ---------------------------------------------------------- |
| `supabase`  | Typed reads, RPC calls matching generated types            |
| `untypedDb` | RPC calls where generated types are outdated or mismatched |

### Rule

- If `typecheck` fails on a `.rpc()` call because generated types are stale, switch to `untypedDb`
- After regenerating types (`supabase gen types`), migrate back to `supabase` when possible

## Render Safety

**Null/undefined guard before render:**

```tsx
// Wrong
<p>{(error as Error).message}</p>

// Correct
<p>{error instanceof Error ? error.message : String(error)}</p>
```

**Stable keys in lists:**

```tsx
// Wrong
items.map((item, i) => <Row key={i} />);

// Correct
items.map((item) => <Row key={item.id} />);
```

**Required states:**

- Loading skeleton when data is fetching — no flash of default values
- Loading guard at correct level (page-level if multiple children share same query)
- Empty state when data is empty array
- Error state with user-readable message
- Long text overflow: use `truncate` or `line-clamp`

**Extract repeated inline styles:**

```tsx
// Wrong — 10 components repeat the same className
<div className="px-4 py-3 rounded-lg bg-[var(--color-emerald-50,#ecfdf5)]">

// Correct — extract to CSS class
<div className="success-inline">
```

## Error Handling

```tsx
// Wrong — swallows error silently
try {
  await mutation();
} catch {}

// Correct — at minimum log, ideally set error state
try {
  await mutation();
} catch (err) {
  const message = err instanceof Error ? err.message : String(err);
  setError(message);
  console.error('[MutationError]', err);
}
```
